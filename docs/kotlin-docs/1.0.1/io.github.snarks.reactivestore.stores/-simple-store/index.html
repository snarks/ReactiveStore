SimpleStore

class SimpleStore<K : Any, V : Any> : ReactiveStore<K, V>

A basic implementation of ReactiveStore

This implementation relies on updateScheduler for its thread safety. So single thread schedulers are preferred if this store will be used in a multithreaded / concurrent environment. Parallel schedulers should be avoided altogether. (The default value, Schedulers.single(), should be adequate in most cases.)

The publishScheduler is an optional scheduler which will be used to emit items from its observable methods
